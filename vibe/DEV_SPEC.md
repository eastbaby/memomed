# 需求背景

我最近发现和我父母交流如何复查疾病的时候很困难，很难找到过往不同医院的各个报告图片，也很难记住如何吃药和如何检查。因此，我想做一个关于管理健康和疾病管理相关的agent。这个agent的前端表现形式是一个对话框，有点类似于豆包/chatgpt应用这种形式。

这个需求同时也是为了我面试做准备，需要用到一些agent主流框架。



# 技术栈选型大纲
后端：Python 3.12 + uv + FastAPI + LangGraph 1.1 + LangChain 1.2。
前端：React + Vite + Tailwind + Lucide 图标。

- LangGraph (推荐)：它是基于 LangChain 的新一代架构。
面试价值：目前面试中，纯 LangChain Chain 已经很普遍，而 LangGraph 强调“有状态的图模型”，能处理复杂的循环逻辑（如：询问不清楚、反思回复、多步骤检查），体现你对 Agent 低层逻辑的深度掌握。
- LangChain：主要的集成工具库，用于调用各种 LLM 和工具。
- ChromaDB (向量数据库)：用于存储父母的过往检查报告片段。
- PostgreSQL (关系型数据库)：存储用户信息、药名清单、提醒设置。


# 0.0.1 版本

## MVP Demo 实现规划 (MVP Implementation)
为了先实现 0.0.1 版本的“最简单 MVP”，我们可以按照以下三个步骤构建：

### 第一阶段：极简聊天链路 (Basic Chat)
目标：打通“用户发送 -> 后端处理 -> AI 回复”的闭环。
功能：前端一个 Chat Window，后端一个 /chat 接口。AI 的身份设定为“Memomed 健康助手”，具备专业的、耐心的沟通风格。
### 第二阶段：文件上传与信息提取 (Simulated Filing)
目标：模拟父母上传报告。
功能：支持上传图片（暂时用 Mock 数据模拟 OCR 提取出的文字），Agent 能够理解报告中的异常指标（如“甘油三酯偏高”），并用大白话解释给父母听。
### 第三阶段：用药记忆与提醒 (Memory)
目标：解决“记不住如何吃药”的痛点。
功能：在对话中通过 Agent 记录“每天吃两片阿司匹林”，Agent 存入数据库，下次询问“我该怎么吃药”时能准确回答。

## 💡 给面试的加分建议
由于你特别提到了是为了面试做准备，在实现这个 MVP 时，我会带你关注以下几个核心工程问题：

Streaming (流式响应)：让用户看到文字逐个蹦出来，提升交互感。
Tool Use (Function Calling)：演示 Agent 如何调用“搜索过往病历”或“添加用药提醒”的函数。
Memory Management：如何管理多轮对话的上下文，尤其是在讨论复杂病史时。
